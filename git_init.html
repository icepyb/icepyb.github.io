初始设置

选个目录 git init *初始化git，会创建一个.git文件夹

全局配置

git config –list 查看所有系统配置项。 git config --global user.email “icepyb@gmail.com” git config --global user.name “icepyb”

配置远程库连接

SSH连接时

ssh-keygen -t rsa -C your_email@youremail.com *建议按三次回车将密钥存在默认位置。因为存在其他地方git好像要经过配置才会使用该密钥，但不会配这个操作。

clip

ssh -T git@github.com -v 可看debug信息 京东为git@code.jd.com *可以测试SSH登陆。

git clone git@ip|domain:user/repo_name.git

HTTPS连接时

由于Github的SSH 22端口登陆总是失败，所以考虑使用GAE代理后的HTTPS方式连接。

git config --global http.proxy “127.0.0.1:8087” *配gae的代理

git config --global http.sslverify false *不进行SSL证书验证

git clone https:\IP|domain\user\ repo_name.git

远端库管理

*在使用git clone 时会默认将目标仓库设为origin，相当于已添加了远端库。

查看远端库

git remote –v *显示为：远端库标签 远端库URL地址，意思是将远端库映射为该标签，以后使用该标签代表该远端库。

添加远端库

git remote add origin git@github.com:yourName/yourRepo.git git remote add origin https://code.jd.com/yourName/yourRepo.git *git remote add远端库标签 远端库URL地址

修改远端库url

git remote set-url origin NEW_URL *git remote set-url远端库标签 新远端库URL地址

远端库操作

提交到远端库

git push origin master *git push 远端库标签 远端库分支 ， 其中origin是典型的标签，master是典型的分支。 *本地仓库会推送到相应远端库的相应分支。

git push origin : *git push 远端库标签 本地库分支名:远端库分支名 *一般当前如果不在该分支时，使用这种方式提交。如果当前在该分支下，也可以直接提交git push

*当git push时可能需要先设置push的类型 git config --global push.default simple git config --global push.default matching

从远端库下载

git pull

更新远程库

git fetch命令，只是更新远程分支上本地没有的数据到本地分支。

合并

git merge

删除远程库分支

git push origin :develop *git push 远端库标签 本地库分支名:远端库分支名，将空的推送到远程分支。
